Rules and Regulations

Submission Guidelines

  • Eligibility: Participants must meet the eligibility criteria specified for the hackathon, including age, skill level, and any other relevant requirements.
  • Team Size: Teams should consist of the maximum number of members allowed, as specified in the hackathon guidelines. No exceptions will be made.
  • Registration: All participants must register in advance through the official registration process. Late registrations may not be accepted.
  • Code of Conduct: Respectful and inclusive behavior is expected from all participants. Harassment, discrimination, or any form of misconduct will not be tolerated.

Eligibility Criteria:

  • Participants must be at least 18 years of age.
  • Participants must be individuals, not teams or organizations, for this hackathon.
  • The hackathon is open to participants worldwide, unless otherwise specified.
  • Each participant must register individually for the hackathon.
  • Team formation, if allowed, should follow the specified guidelines regarding team size and composition.
  • Participants should have access to a stable internet connection throughout the event.
  • Participants must have the necessary hardware and software required for coding and project development.
  • Basic knowledge of programming and AI concepts is recommended but not mandatory.
  • All participants are expected to follow the event's Code of Conduct.
  • Any violation of the Code of Conduct may lead to disqualification and removal from the event.
  • Previous winners of the same hackathon may not be eligible to participate, depending on the event's rules.
  • Judges, organizers, and their immediate family members are not eligible to participate in the hackathon.
  • Projects that have been published as open source or have won prizes in other competitions may have specific eligibility requirements.
  • Participants are usually allowed to submit only one project. If multiple entries are allowed, they should follow the specified guidelines.
  • Participants affiliated with the hackathon's organizers or sponsors may have special eligibility considerations. Check the rules for details.
  • Certain hackathons may have additional eligibility criteria or restrictions, which will be clearly stated in the event guidelines.

Project Submission Rules:

  • Submission Deadline - Projects must be submitted before the specified deadline; late submissions may not be accepted.
  • Team Collaboration - If team participation is allowed, all team members must be registered participants of the hackathon.
  • Originality - Projects should be entirely original and created during the hackathon. Plagiarism or using pre-existing projects is not allowed.
  • Project Documentation - Participants must provide comprehensive project documentation or a README file explaining the project's purpose, how to run it, and any dependencies.Project Documentation - Participants must provide comprehensive project documentation or a README file explaining the project's purpose, how to run it, and any dependencies.
  • Code Repository - If applicable, project code should be hosted on a specified platform (e.g., GitHub), with a provided link.
  • Video Presentation - A brief video presentation or demo of the project may be required; guidelines for format and length should be followed.
  • Submission Format - Submissions should follow the specified format, including file types and naming conventions.
  • Source Code Sharing - In some cases, participants may be required to share their project's source code with the organizers for evaluation.
  • Intellectual Property - Participants typically retain ownership of their projects, but organizers may have limited rights for promotion and display purposes.
  • Multiple Submissions - Clarify whether participants can submit multiple projects and any restrictions on doing so.
  • Accessibility - Projects should be accessible and usable by judges for evaluation. Provide clear instructions for access and usage.
  • Language - Project documentation and presentation should be in a specified language or format if required.

House Rules:

  • Respect and Politeness: Treat all participants, judges, and organizers with respect and politeness. Harassment, discrimination, or disrespectful behavior will not be tolerated.
  • Code of Conduct: Adhere to the event's Code of Conduct, which outlines the expected behavior and consequences for violations.
  • Cooperation: Encourage cooperation and collaboration among participants. This is a learning experience, and sharing knowledge is encouraged.
  • Safe Environment: Create and maintain a safe and inclusive environment where everyone feels comfortable and valued.
  • No Cheating: Cheating, including but not limited to plagiarism, using unauthorized resources, or unauthorized collaboration, is strictly prohibited.
  • Fair Play: Follow the rules and guidelines of the hackathon to ensure fair competition.
  • Good Sportsmanship: Display good sportsmanship whether you win or lose. Respect the judges' decisions and fellow participants.
  • No Mishandling of Data: Ensure the ethical and responsible handling of data. Respect privacy and confidentiality.
  • Abide by Laws: Comply with all local, national, and international laws and regulations.
  • Responsible Use of AI: If AI models or tools are provided, use them responsibly and avoid generating harmful or inappropriate content.
  • Reporting Violations: If you witness any violations of these rules, report them to the event organizers promptly.
  • Consequences: Violating these house rules may lead to disqualification and removal from the hackathon. Organizers have the authority to enforce consequences as deemed necessary to maintain a respectful and fair environment.
